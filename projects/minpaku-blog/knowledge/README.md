# 知識ベース（RAG）

このフォルダの `.md` ファイルは、**記事生成時に Claude API へ自動投入**されます。
ここを充実させるほど、生成記事の質と独自性が上がります。

## ファイル構成

| ファイル | 役割 | 充実度の目安 |
|---|---|---|
| `voice_guide.md` | ブランドボイス・トーン・避けたい表現 | 必須・最初に作る |
| `target_audience.md` | ターゲット読者の人物像（年齢・課題・目的） | 必須・最初に作る |
| `industry_facts.md` | 民泊業界の事実集（法規制・市場データ・用語定義） | 推奨・徐々に増やす |
| `personal_experiences.md` | 運営者本人の実体験・失敗談・ノウハウ | **最重要・人間超えの源泉** |
| `unique_perspectives.md` | 他ブログにない切り口・独自視点 | 推奨 |
| `past_articles/*.md` | 過去のベスト記事（few-shot例として使う） | 充実すると劇的に質UP |

## 運用ルール

- **空でも動きます**（プロンプトはファイルが空のときは読み飛ばす設計）
- **少しずつ追加**してOK。記事を書きながら気づいた知見を `personal_experiences.md` に追記する運用を推奨
- **過剰に詳細化しない**。冗長になると逆にプロンプトが薄まる
- **目安**: 全部合わせて 30,000 トークン（≈ 50,000 文字）まで
  - これを超えたら ChromaDB 等のベクトル検索を導入する

## プロンプトキャッシュ

これらのファイルは **Claude API のプロンプトキャッシュ**に乗るため、知識ベースを大きくしてもコストはほぼ増えません（2回目以降の呼び出しでキャッシュヒット）。
